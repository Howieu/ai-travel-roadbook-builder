# AI Travel Roadbook Builder V0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new GitHub repo containing a reusable Codex skill plus a static web roadbook renderer/demo for turning pasted travel materials into `roadbook.json` and a shareable HTML roadbook.

**Architecture:** Keep V0 static and deterministic. The skill guides the agent to extract and structure trip data; a small Python renderer validates the JSON and generates the web page; the static site under `docs/` hosts the product page and generated demo for GitHub Pages.

**Tech Stack:** Codex skill folder, Markdown references, Python 3 standard library, static HTML/CSS, GitHub Pages.

---

## Task 3: Build Renderer And Demo Data

- [ ] **Step 1: Implement Python renderer**

Renderer requirements:

- stdlib only;
- read JSON input;
- check required fields: `trip.title`, `trip.destination`, `days[].date`, `days[].stops[].name`;
- HTML-escape user-provided text;
- generate Amap, Google Maps, and Apple Maps search links from `mapQueries`;
- render lodging, transport, warnings, day cards, stop cards, deadlines, fallbacks, source, confidence.

- [ ] **Step 2: Add demo JSON and pasted-material sample**

Use a Paris demo with hotel, arrival flight, return train, two days, and stops with mixed confidence labels.

- [ ] **Step 3: Generate demo HTML**

Run:

```bash

```
